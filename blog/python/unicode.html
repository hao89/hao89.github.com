Something that confuse me a lot at first is that I thought Unicode and UTF-8 are the same. Actually they are different. UTF-8 is comparable to ASCII, UFT-16,.etc. They represent different ways of storing (encoding) strings in memory. While Unicode, in comparison, is the plaintext (明文). A character's unicode is a representing number of that character in the unicode world, and that's it and no more.

If we have a unicode of a character, we can use utf-8 to encode the character and we can get the exact value we used to store that character. On the contrary, if we know how a character is stored in bytes, we can decode that to unicode given its encoding method ('utf-8 or 'ascii'?).
